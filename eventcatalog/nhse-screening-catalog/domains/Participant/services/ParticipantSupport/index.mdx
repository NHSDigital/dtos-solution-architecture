## Overview

The ParticipantSupport service is responsible for managing non-digital participant interactions and support requests. It handles inbound calls, manages participant queries, and ensures participants can access support through various channels.

## Responsibilities

1. **Call Management**

   - Handle inbound support calls
   - Route calls to appropriate agents
   - Track call outcomes and resolutions
   - Maintain call quality standards

2. **Query Management**

   - Process participant queries
   - Track query status and resolution
   - Escalate complex queries
   - Maintain query history

3. **Support Provision**
   - Provide non-digital support options
   - Manage support appointments
   - Handle accessibility requests
   - Coordinate with other services

## Service Interactions

### Receives Events

- `ParticipantCreated`: When a new participant is registered
- `ParticipantUpdated`: When participant information is updated
- `SupportRequestCreated`: When a new support request is created
- `SupportRequestUpdated`: When a support request is updated

### Sends Events

- `SupportCaseCreated`: When a new support case is created
- `SupportCaseUpdated`: When a support case is updated
- `SupportCaseResolved`: When a support case is resolved

## Business Rules

1. All support cases must be associated with a valid participant
2. Support cases must be assigned to qualified agents
3. Call quality standards must be maintained
4. Accessibility requirements must be met
5. Support cases must be resolved within defined timeframes
6. Escalation procedures must be followed for complex cases
